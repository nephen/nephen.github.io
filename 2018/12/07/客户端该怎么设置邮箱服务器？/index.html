客户端该怎么设置邮箱服务器？

发表于 2018-12-07 | 分类于 技术杂记 | | 本文总阅读量 次

最近发现一个问题，苹果手机在设置邮箱服务器来获取第三方邮件时总是报错，无法正常获取邮件，我思考了一下，这其中肯定是有原因的，不可能是手机软件的的问题，于是决定研究一下邮箱服务器的一些原理，解决这个看起来很基础的问题。

整体架构

所谓邮箱服务器即为网络上的客户端保存电子邮件（电子邮件）消息的远程或中央计算机称为邮件服务器。


电子邮件发送协议是一种基于“ 推 ”的协议，主要包括 SMTP；邮件接收协 则是一种基于“ 拉 ”的协议，主要包括POP协议和IMAP协议。邮件发送和接收的流程如上图所示，其中用户代理UA（User Agent ）是一个软件包（程序），它的功能是：撰写、阅读、回复和转发报文，还能处理邮箱（如创建收信箱和发信箱）。共有两种类型的用户代理： 命令驱动 和GUI 。 命令驱动 属于早期的电子邮件，通过命令发送和接收邮件，而GUI 则是包含图形界面，允许用户使用键盘和鼠标与软件进行交互。

下面来详细看一下用户发送与邮件接收的过程：

  1. 当发件人单击“发送”按钮时，电子邮件客户端将使用SMTP协议连接到电子邮件提供商的SMTP服务器（邮件传输代理）。假设您使用的是Gmail，那么您的电子邮件客户端会联系Gmail的SMTP服务器 - smtp.gmail.com。 Gmail上的SMTP服务器会检查您的电子邮件中的收件人电子邮件地址。假设收件人是 person@example.com，则Gmail SMTP服务器会从DNS查找example.com的MX（邮件交换器/Mail eXchange）记录。 MX记录是DNS中的一条记录，用于指定负责接收方域接受电子邮件的邮件服务器。从MX记录中检索example.com的SMTP服务器地址后，Gmail SMTP服务器会将该电子邮件发送到example.com的SMTP服务器。 example.com的SMTP服务器检查该服务器上是否存在“person”收件人。如果该服务器上存在该帐户，则会将该电子邮件转发到其自己的IMAP / POP3服务器（邮件传递代理）以存储此电子邮件。

  2. 现在，收件人可以使用Outlook（邮件用户代理）等电子邮件客户端连接到此服务器并阅读该电子邮件。有时在发件人和收件人的SMTP服务器之间有多个站点，整体上流程通常是相同的。
